Easy Fuel · Site TSTSTE0001 · Portal https://portal.easyfuel.ai

  • IsNotVerifiedResponse
  • IsAccountDecryptionKeyMissing

Process: getTestConfiguration → setTestConfiguration (toggle flag) → mock/v1/requestpayout → mock/v1/getpayout.

Easy Fuel: No mock API integration in the application. These three tests must be run with curl/Postman using your SiteCode and Payout ApiKey, then JSON responses emailed to Itu.

Status: Not in app — manual only (expected for sign-off)

3. Request body – Itu example vs our code

Itu / Ozow staging expects (example):

{ "siteCode": "TSTSTE0001", "amount": 0.1, "merchantReference": "yourReference", "customerBankReference": "yourReference", "isRtc": false, "notifyUrl": "https://portal.easyfuel.ai/api/webhooks/ozow-payout-notification", "bankingDetails": { "bankGroupId": "3284a0ad-ba78-4838-8c2b-102981286a2b", "accountNumber": "0000000035", "branchCode": "632005" }, "hashCheck": "..." }

Easy Fuel submitOzowPayout() sends today:

{ "siteCode": "...", "amount": "123.45", "currency": "ZAR", "reference": "...", "notificationUrl": "...", "verificationUrl": "...", "bankAccountName": "...", "bankName": "...", "accountNumber": "...", "branchCode": "...", "accessToken": "..." }

Gaps to fix before live payout tests through the app:

  • Missing merchantReference, customerBankReference, isRtc
  • Missing bankingDetails.bankGroupId (we store bank name, not Ozow UUID)
  • Missing hashCheck (payout hash algorithm not implemented)
  • notifyUrl vs notificationUrl — confirm exact field names with Itu
  • OZOW_PAYOUT_API_KEY not set in .env (code falls back to Client Secret — likely wrong for payouts)
  • OZOW_PAYOUT_DRY_RUN=true skips all Ozow payout API calls

4. What is already correct

  • Staging base URL default: https://stagingpayoutsapi.ozow.com
  • Submit path default: /v1/requestpayout
  • Headers: SiteCode + ApiKey on POST
  • Verification URL: https://portal.easyfuel.ai/api/webhooks/ozow-payout-verification
  • Notification URL: https://portal.easyfuel.ai/api/webhooks/ozow-payout-notification
  • Access token (24 chars): configured as OZOW_PAYOUT_ACCESS_TOKEN
  • DB tables: payment_transactions, payout_transactions
  • Pay-in → payout trigger after customer payment completes (driver net amount)
